## iMessage bug found + fixed today

Symptom Dave reported: texting Bishop's number got no reply, but the messages were visibly arriving in Messages.app on the Mini.

Root cause (from `/tmp/openclaw/openclaw-2026-06-30.log`): `channels.imessage.cliPath` in `openclaw.json` was set to `/opt/homebrew/lib/node_modules/openclaw/skills/imsg` — that's the **skill folder** (just `SKILL.md`, not executable), not the actual `imsg` binary (`/opt/homebrew/bin/imsg`, installed via `brew install steipete/tap/imsg`). Fix applied:
```
openclaw config set channels.imessage.cliPath /opt/homebrew/bin/imsg
openclaw gateway restart
```
Confirmed in logs: `[default] starting provider (/opt/homebrew/bin/imsg)` with no follow-up errors, `openclaw channels status` shows `iMessage default: enabled, configured, running`.
